How to fix "replacement transaction underpriced"
This error means a previous transaction is stuck in the mempool at the same nonce.
The gateway now auto-retries with a 15% gas price bump on each attempt.

If transactions are still failing:
1. Click Check pending nonce — if "stuck count" > 0, proceed to step 2.
2. Click Cancel stuck transactions — sends zero-value self-transfers to flush the mempool.
3. Wait ~30 s for cancellations to confirm, then retry your original action.

Alternatively on the command line:
curl -X POST http://<gateway>:8000/admin/cancel-pending
